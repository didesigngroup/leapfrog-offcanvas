Typography

Typeface & Color

Arial is the default typeface with a line-height of 1.42857143 (10:7). If the Finacial Institution has selected a widget heading font in the Admin Platform tool then Heading 1 will take that font.

Font color are Hex #333333 or #666666. These font colors guarantee compliance with WCAG AA accessibility guidelines if used on a white background or on a gray call out Hex #ededed.

Heading 1 Arial 28px #333333, line-height 40px

Heading 2 Arial 24px #333333, line-height 34px

Subheading Arial 18px #333333, line-height 26px

Main font size Arial 16px #333333 or Arial #666666, line-height 23px

For long paragraph Arial 14px #666666, line-height 20px

Legal text Arial 13px #666666, line-height 19px

Links (#0055CC in case of FI then FI Color) inherit the size of body font size.

Success Heading Arial 24px #2E6F03
Success messages Arial 14px #2E6F03
Error messages 14px #C90202

Icons

How to use

Icons fall into 2 categories, they either indicate that the user can take an action or they draw the attention of the user to an area or message. Icons should only be used for the meanings indicated below, if they are used in other ways the meaning of the icon will be lost to the user.

If the icon is not universally known then it must have a description associated with the image. When space/hierarchy permit icons used as buttons should have some kind of button design to indicate that the icon can be tapped to take an action.

Color

The Financial Institution can choose a link/icon color in the AP branding tool. That color should meet the WCAG contrast ratio for AA compliance with accessibility guidelines. Test color compliance at this site:

http://snook.ca/technical/colour_contrast/colour.html

If the Financial Institution did not select a link/icon color, then the icon must use one of the HEX values below to meet accessibility guidelines:

#333333, #666666, or #FFFFFF (if on a dark background).

Accessibility

All images must have an alt attribute (even if the function of the image is purely decorative and alt=””). When an image has meaning to a sighted user, convey this with the alt attribute if possible. Avoid images that contain text; if they do, the alt value must contain the same string of text.

Responsive UI

Breakpoint

We use fixed containers driven by the device width. Use default Bootstrap breakpoints (XS,S,M,L) along with the container width unless specified differently for a design. Page background is #EDEDED. Container background is #FFFFFF.

Device width Container width
(L) 1200px or higher 1170px
(M) 992px to 1199px auto
(S) 768px to 991px auto
Less than 768px auto

Mixed: mobile and desktop

The Bootstrap 3 grid system has four tiers of classes: xs (phones), sm (tablets), md (desktops), and lg (larger desktops). You can use nearly any combination of these classes to create more dynamic and flexible layouts.

Each tier of classes scales up, meaning if you plan on setting the same widths for xs and sm, you only need to specify xs.

Padding

Primary Container / Panel

Primary Container will follow the following padding guideline on each breakpoint. What is excepted by the user is to apply class name 'first' on each column top most 'di-container'. This is to follow the spec. to have right value of top-padding on first container of each column.

Breakpoint Padding (from the edge of the screen to content)
L 30px all side, bottom 40px.
M 30px all side, bottom 40px.
S 20px all side, bottom 40px.
XS 20px all side, bottom 40px.
